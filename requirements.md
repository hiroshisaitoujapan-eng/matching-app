# 要件定義書

## プロジェクト概要

Tinder 風のシンプルなマッチングWebアプリ。スワイプUIで異性とマッチングし、チャットで交流できる。

## 技術スタック

| 役割 | 技術 |
|------|------|
| フロントエンド | Next.js (App Router) + TypeScript + Tailwind CSS |
| バックエンド・DB | Supabase (Auth / PostgreSQL / Realtime / Storage) |
| ホスティング | Vercel |
| バージョン管理 | GitHub |

## 対象ユーザー

- 性別：男性・女性の2択
- マッチング対象：異性のみ

---

## 機能要件

### 1. ユーザー管理

#### 1-1. 会員登録
- メールアドレス + パスワードで登録
- 登録後、プロフィール設定ページへ誘導

#### 1-2. ログイン・ログアウト
- メールアドレス + パスワードでログイン
- セッション管理は Supabase Auth に委譲
- ログアウト機能

#### 1-3. プロフィール設定
- 必須項目：ニックネーム、生年月日（年齢計算）、性別、自己紹介文
- 任意項目：居住地、趣味（タグ選択）
- プロフィール写真：最大5枚アップロード（Supabase Storage）
- プロフィール編集（設定後も変更可能）

### 2. マッチング体験

#### 2-1. カード表示
- 異性ユーザーのプロフィールカードを1枚ずつ表示
- カード内容：写真・ニックネーム・年齢・居住地・自己紹介文
- 写真はタップ/クリックで切り替え

#### 2-2. スワイプ操作
- 右スワイプ or「いいね」ボタン：いいね送信
- 左スワイプ or「スキップ」ボタン：スキップ（その日は再表示しない）
- スワイプアニメーション付き
- 表示候補がなくなった場合のEmptyState表示

#### 2-3. マッチング判定
- 双方が互いにいいねした場合にマッチング成立
- マッチング成立時：祝福モーダルを表示
- マッチング成立後：チャット画面へ誘導

### 3. 交流その他機能

#### 3-1. テキストチャット
- マッチングしたユーザー間のみメッセージ送受信可能
- リアルタイム更新（Supabase Realtime）
- 既読・未読管理
- メッセージ一覧（チャットルーム一覧）

#### 3-2. 通知
- マッチング成立時の通知（アプリ内バナー）
- 新着メッセージの通知（アプリ内バナー）
- ブラウザのプッシュ通知（Web Push API）

#### 3-3. ブロック・通報
- ユーザーをブロック：ブロックしたユーザーはスワイプ候補・チャット一覧から除外
- ユーザーを通報：カテゴリ選択（迷惑行為・不適切な写真・スパム・その他）+ 詳細テキスト
- 通報データは管理用テーブルに保存

## 非機能要件

### セキュリティ
- 認証が必要なページは全て保護（未認証ならログインページへリダイレクト）
- Row Level Security (RLS) を Supabase で設定し、他ユーザーのデータを直接取得・変更できないようにする
- プロフィール写真はプライベートバケットで管理、署名付きURLで配信

### パフォーマンス
- カード表示は次の数件を事前フェッチ（プリフェッチ）
- 画像は Next.js Image コンポーネントで最適化

### 開発・運用
- GitHub リポジトリで管理
- `main` ブランチへのマージで Vercel へ自動デプロイ
- `develop` ブランチを開発ベースとし、機能ごとに feature ブランチを切る
- PR を通じたコードレビューフロー

---

## 対象外（スコープ外）

- ビデオ通話
- 有料プラン・課金機能
- 管理者画面
- Android / iOS ネイティブアプリ
- LGBTQ+ 対応
